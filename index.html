Overview

    General API Information:

        Base URL for BBD: https://{BaseURL}/api/ (request the BaseURL from the partner)
        Version : v1.0.1
        Last Update: 25th, April 2024

LivreurDeliveryAPI is an API that works on optimizing the delivery process as well as enabling partnership between delivery companies.

Important Definitions:

  • Order/parcel : a package or item that is being trasported from a location to another.
  • Delivery Company: an organization responsible of trasporting parcels from customers(sellers) to the receivers(buyers).
  • Order Tracking: provides the customers updates on the current location, expected delivery time, and any relevant status changes of their orders.
  • Distribution Note: a note that contains the basic information of all the parcels sent to a specific delivery man for transforming it to final receiver.
  • Payment Note: contains all the parcels that receivers got & payed to delivery man, with detailed information about the payment.
  • Sent Note: contains the parcels received by a specific zone for delivery.


Description:

This API facilitates collaboration between Company_A, operating in Zone_A, and Company_B, located in Zone_B, allowing the first to efficiently send parcels for distribution within Zone_B.
With this the API enables companies to respond quickly to customer needs while minimizing expenses and efforts associated with parcel distribution.

    * EndPoints
  • The API provides many endpoints either endpoints related to managing orders, sent notes, distribution notes, and payment notes.
    * Request & reponse
  • The API utilizes both POST and GET methods for sending HTTP requests and responds with JSON data.

/changeParcelStatus



# change the status of a parcel query example
curl --request POST \
--url http://{BaseUrl}/api/changeParcelStatus \
--header 'Accept: application/json' \
--header 'Content-Type: application/json' \

Post required data to edit the order's status.
This endpoint is for changing the status of an order using the API. 



    
---Success Result example (Change the status of a parcel):
{
    query:{
        "user_token":"7vWc9QBKpTzZoT25lmmvtKM93JGTYpknX2VHOPP", 
        "bar_code": "ar092364856CB",
        "status_code": "v",
        "status_comment": "order information to be verified",
        "status_date": 0
    }
    result: [
        {
            "message": "order status fetched successfully",
            "bar_code":"ar092364856CB",
            "status":200,
            "success": true
        }
    ]
}
query parameters
Field Type Description
user_token String Required It's the API key specified for a unique user (delivery man), ask your partner for it.
bar_code String Required The order's barcode.
status_code String Required The Code of the new status of the order. Example: "DELIVERED".
status_comment String Required For the parcels that have a required comment, a comment needs to be added from here.
status_date String Required It's the status date for the status that requires it. Example, "REPORTED" status needs the date to where the delivery is reported.

name String The names of the folders.
dates String The date of reporting (to which date the parcel's process was reported).
